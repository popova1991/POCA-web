Junior

!

Важно понимать: любые квалификационные рамки относительны. Приведенная информация обобщает распространенную практику, но не отменяет внутренних матриц компетенций. Так, одним отделам не требуются навыки работы со снифферами или SQL-запросами, однако взамен необходимо углубленное владение разными типами API. Другим же подразделениям не нужно тестировать бэкенд, но предъявляются повышенные требования к проверке интерфейсов. Рекомендуем воспринимать данный материал как путеводитель, а за актуальными уровнями грейдов обращаться к правилам конкретной компании.

Уровень — поддержка

Выполняет только под контролем, требует проверки

Сбор требований Глубинные интервью Присутствует, задаёт 1–2 уточняющих вопроса по готовому списку, не ведёт сам
Сбор требований Работа с неопределённостью Не начинает работу, пока требование не описано дословно
Документирование User Story Заполняет фиксированный шаблон («Как <роль>, я хочу…»)
Документирование Техническое задание Не пишет, только читает готовое
Документирование Диаграммы Копирует стиль наставника, рисует только Sequence

Управление требованиями Трассировка Отслеживает связи «задача → баг» в Jira по инструкции
Проектирование Модель данных Рисует таблицу из 3–5 полей, типы данных (копирует)

Уровень — система

Создаёт стандарты, масштабирует практики

Интервью: «создал ли стандарт, который используют другие?»

Факт выполнения Компетенция Индикатор Что делает
Сбор требований Глубинные интервью Управляет группой 5+ человек, снимает групповое давление
Сбор требований Работа с неопределённостью Создаёт прототип за 1 день для снятия неопределённости
Сбор требований Эвристики Использует обратную мозговую атаку (выявление рисков)
Документирование User Story Создаёт шаблоны Story для разных типов задач
Документирование ТЗ Создаёт Living Documentation (документация как код)
Документирование Диаграммы Внедряет C4 model для всей системы, проверяет консистентность
Управление требованиями Приоритезация Создаёт модель приоритезации для продукта с весами
Управление требованиями Управление изменениями Создаёт Impact Assessment Checklist для команды
Проектирование Модель данных Создаёт Data Vault или Anchor Model для больших данных
Проектирование API Внедряет GraphQL или gRPC по критериям
Проектирование Интеграции Внедряет паттерны: Saga, Outbox, Circuit Breaker
Проектирование НФТ Внедряет Chaos Engineering для проверки НФТ
Коммуникации Управление ожиданиями Договаривается о замене одной фичи на другую
Коммуникации Конфликт требований Находит компромиссный вариант (третье решение)
Коммуникации Фасилитация Проводит EventStorming на 20 человек
Технические навыки SQL EXPLAIN ANALYZE, оптимизация запросов
Технические навыки API-тесты Пишет автотесты на коллекции (Newman)
Технические навыки Чтение кода Делает Code Review на логику (не на синтаксис)
Технические навыки Логи Строит дашборд по логам (частота ошибок)
Менторство Передача знаний Ведёт внутренний курс (4+ часов)
Менторство Адаптация Проводит buddy-программу (2 месяца)
Оценка и риски Оценка SP Даёт диапазон: оптимистичную и пессимистичную
Оценка и риски Риски Предлагает mitigation и fallback план
