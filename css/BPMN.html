BPMN

Цели моделирования

Различия: для согласования с бизнесом / для передачи разработчикам / для регламента / для анализа узких мест.

Уровни абстракции

Разница между Descriptive (кто что делает), Analytical (логика и данные), Executable (BPMS, автоматизация).

Золотое правило BPMN

Один процесс — одна диаграмма. Если не влезает на A3/экран, значит, вы неправильно декомпозировали.

Потребители диаграммы

Кто будет читать: бизнес-пользователь (нужны пулы и роли), разработчик (нужны события и данные), тестировщик (нужны альтернативные потоки).

Пулы и дорожки (Pool & Lane)

Разница между пулом (сущность, внешняя система) и дорожкой (роль внутри пула). Вертикальное vs горизонтальное расположение.

События начала и конца

Start Event (один процесс — одно стартовое), End Event (может быть несколько). Типы: message, timer, none.

Задачи (Tasks)

User Task (человек), Service Task (система), Script Task (автоматический скрипт), Business Rule Task (вызов правил).

Потоки управления (Sequence Flow)

Правила соединения: нельзя соединять пулы напрямую (нужны сообщения), нельзя вести поток из пула в лейн.

Простая логика ветвления

Exclusive Gateway (XOR) — «или-или». Как правильно ставить условия, формат выражений.

Параллелизм

Parallel Gateway — разветвление и схождение. Правило: сколько входов, столько выходов должно сойтись.

Промежуточные события (Intermediate Events)

Timer (ждать), Message (получить ответ), Error (ошибка в процессе), Signal (шина событий).

Сообщения (Message Flow)

Как правильно показать коммуникацию между пулами. Message Flow всегда пунктирный, пересекает границы пулов.

Сложные гейты

Inclusive Gateway — «и/или» (выполняется один или несколько путей). Event-Based Gateway — «кто первый» (реакция на событие).

Вложенные процессы (Sub-process)

Collapsed vs Expanded. Когда выносить в подпроцесс. Loop Characteristics (повторение).

Компенсация (Compensation)

Как моделировать откат действий. Compensation Boundary Event, Compensation Task.

Артефакты (Data Objects, Group, Annotation)

Как показывать данные (документы, записи в БД). Группировка для читаемости.

Анализ текста требований

Техники вырезания: вычленяем действующих лиц, события, шаги, условия, исключения.

Выделение границ процесса

Что такое «хороший» старт (триггер) и «хороший» конец (выходные критерии). Избегать «вечных» процессов.

Выделение пулов и дорожек

Как не перепутать роль и систему. Когда нужен отдельный пул для внешней системы.

Работа с исключениями

Happy Path vs Unhappy Path. Как не превратить диаграмму в паутину.

Декомпозиция

Правила: уровень детализации = один исполнитель за один шаг. Когда использовать Sub-process.

Проверка модели на валидность

Чек-лист: все пути закрыты, нет «висячих» стрелок, все гейты имеют выходы, все пулы взаимодействуют через Message Flow.

Моделирование с группой (EventStorming + BPMN)

Как проводить сессию: сначала стикеры (действия, события), потом формализация в BPMN.

Согласование модели со стейкхолдерами

Как презентовать диаграмму бизнесу (скрыть технические детали, показать роли). Как собирать обратную связь.

Версионирование и хранение

Где хранить BPMN (репозиторий, wiki). Правила именования файлов. Связь с требованиями (Jira/Confluence).

BPMN для разработки

Как передать модель разработчикам: что такое «исполнимая» BPMN (Camunda, Zeebe). Где нужны Service Task, где — User Task.

BPMN для тестирования

Как генерировать тестовые сценарии из диаграммы: каждый путь — отдельный тест-кейс.

Анализ метрик процесса

Как с помощью BPMN найти узкие места: количество гейтов, длина путей, точки ручного вмешательства.

Что такое BPMN

BPMN (Business Process Model and Notation) — это стандарт для моделирования бизнес-процессов. Он позволяет одинаково понимать процессы аналитикам, разработчикам и бизнесу.

Элементы BPMN

Основные элементы: события (events), задачи (tasks), шлюзы (gateways), потоки (flows).

Примеры

Пример: процесс оформления заказа, процесс авторизации пользователя и др.
